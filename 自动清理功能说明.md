# 自动清理功能说明

## ✅ 功能状态：已完成并集成

自动清理功能已集成到 `test_cases/conftest.py`，测试结束后会自动清理跟踪的ID。

---

## 🎯 如何使用

### 1. 在测试函数中添加 `db_cleanup` 参数

```python
def test_create_counterparty(counterparty_api, db_cleanup):
    #                                        ^^^^^^^^^ 添加这个参数
```

### 2. 创建资源后调用 `track_created_id()`

```python
from test_cases.conftest import track_created_id

def test_create_counterparty(counterparty_api, db_cleanup):
    """创建Counterparty - 会自动清理"""
    
    # 创建
    response = counterparty_api.create_counterparty(
        recipient_name="Auto TestYan Counterparty 001",  # ⚠️ 必须前缀
        email_address="auto.testyan@example.com",
        ...
    )
    
    # 提取ID
    created_id = response.json()["data"]["id"]
    
    # ⚠️ 关键：跟踪ID
    track_created_id("counterparty", created_id)
    
    # 继续测试...
```

### 3. 测试结束后自动清理

当所有测试运行完成后（session结束），`db_cleanup` fixture 会自动：
1. 遍历所有跟踪的ID
2. 调用清理方法删除数据
3. 记录清理统计

## 🔧 工作原理

### 跟踪阶段（测试运行时）

```python
# 测试1创建了counterparty
track_created_id("counterparty", "id_001")

# 测试2创建了contact
track_created_id("contact", "id_002")

# 测试3创建了sub_account
track_created_id("sub_account", "id_003")

# 内部存储结构：
test_created_ids = {
    "counterparty": ["id_001"],
    "contact": ["id_002"],
    "sub_account": ["id_003"],
    ...
}
```

### 清理阶段（测试结束后）

```python
# db_cleanup fixture 自动执行：
for module, ids in test_created_ids.items():
    if ids:
        cleaner.cleanup_by_ids(module, ids, dry_run=False)
        # 删除 counterparty: id_001
        # 删除 contact: id_002
        # 删除 sub_account: id_003
```

## ⚠️ 重要提醒

### 1. 必须添加 db_cleanup 参数

```python
❌ 错误
def test_create_something(counterparty_api):
    # 无法使用自动清理

✅ 正确
def test_create_something(counterparty_api, db_cleanup):
    # 可以使用自动清理
```

### 2. 必须调用 track_created_id()

```python
❌ 错误
response = api.create(...)
created_id = response.json()["data"]["id"]
# 忘记跟踪，不会自动清理

✅ 正确
response = api.create(...)
created_id = response.json()["data"]["id"]
track_created_id("counterparty", created_id)  # 会自动清理
```

### 3. 数据名称必须以 "Auto TestYan" 开头

```python
❌ 错误
recipient_name="Test Counterparty"  # 不会被清理

✅ 正确
recipient_name="Auto TestYan Counterparty 001"  # 会被清理
```

### 4. 数据库必须配置

检查 `.env` 文件中的数据库配置是否正确。
