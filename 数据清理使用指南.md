# 数据清理使用指南

## 📋 可用的清理脚本

### 1. cleanup_counterparty.py
**清理模块**: Counterparty + Counterparty Group

**执行命令**:
```bash
cd "/Users/mac/api auto/api_auto_framework"
python3 cleanup_counterparty.py
```

**清理内容**:
- actc.t_share_recipient（收款人）
- actc.t_share_recipient_account_relation（收款人账户关联）
- actc.t_share_recipient_group_relation（收款人组关联）
- actc.t_share_recipient_group（收款人组）

### 2. cleanup_contact_sub_fbo.py ⭐ 新增

## 💾 备份机制

### 备份表命名规则
```
原表: actc.contact
备份: actc.contact_backup_20260209_154530
```

### 备份内容
只备份名称以 "Auto TestYan" 开头的数据

### 查看备份表
```bash
python3 restore_from_backup.py
```

## 🔄 恢复数据

### 自动查看备份
```bash
python3 restore_from_backup.py
```

### 手动恢复（SQL）
```sql
-- 恢复Contact数据
INSERT INTO actc.contact
SELECT * FROM actc.contact_backup_20260209_154530;

-- 删除备份表（恢复后可选）
DROP TABLE actc.contact_backup_20260209_154530;
```

## ⚠️ 安全提醒

1. **只删除测试数据** - 名称必须以 "Auto TestYan" 开头
2. **先备份后删除** - 备份失败则终止清理
3. **需要确认** - 必须输入 'yes' 才执行
4. **保留备份** - 建议保留备份表24小时
5. **定期清理** - 建议每周清理一次

## 🎯 日常使用建议

### 每周清理一次
```bash
# 1. 清理 Counterparty
python3 cleanup_counterparty.py

# 2. 清理 Contact/Sub Account/FBO
python3 cleanup_contact_sub_fbo.py
```

### 保留备份24小时
清理完成后，备份表会自动创建。建议24小时后再手动删除备份表。

### 删除备份表
```sql
-- 查看所有备份表
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'actc' AND table_name LIKE '%_backup_%';

-- 删除指定备份表
DROP TABLE actc.contact_backup_20260209_154530;
```

## 📝 常见问题

### Q: 备份表会占用多少空间？
A: 取决于测试数据量，通常几MB到几十MB。

### Q: 备份表会自动删除吗？
A: 不会，需要手动删除。建议24小时后删除。

### Q: 如果备份失败怎么办？
A: 脚本会自动终止清理，不会删除任何数据。

### Q: 能否不创建备份？
A: 当前版本强制备份，保证数据安全。
